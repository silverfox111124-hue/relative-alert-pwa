# 相対アラート PWA (v0.1)

Android Chrome 向けの軽量 PWA 試作品です。サーバー不要・ログイン不要・外部ライブラリなしで動作します。

## v0.1 機能

- 基準時刻入力 (`HH:mm`。例: `08:20`, `9:05`)
- プリセット選択
- `offsetMinutes` による相対時刻計算（マイナス / 0 / プラス対応）
- 計算結果プレビュー（時刻順）
- 通知許可リクエスト
- 通知テスト（10秒 / 20秒 / 30秒）
- セット中アラート一覧表示（v0.1では主に通知テストのセット状況を表示）
- すべてキャンセル
- `localStorage` 保存（基準時刻・選択プリセット中心）
- PWA 基本構成（manifest + service worker）

## 使い方

1. `index.html` を Android Chrome で開きます（ローカルサーバー推奨）。
2. 基準時刻を入力し、プリセットを選択します。
3. 「計算してプレビュー」で一覧を確認します。
4. 「通知許可を確認」で通知を許可します。
5. 10秒 / 20秒 / 30秒テスト通知で実機確認します。
6. 必要に応じて「すべてキャンセル」を押します。

注意：v0.1では、計算結果プレビューから実アラートを一括セットする機能はまだありません。10秒 / 20秒 / 30秒テスト通知は、PWA通知が端末で動くか確認するためのものです。

## 動作確認方法

### PC（簡易）

```bash
python3 -m http.server 8080
```

`http://localhost:8080` を開いて、UI と時刻計算を確認します。

### Android Chrome

1. 同一ネットワーク上の端末からページを開きます。
2. Chrome メニューから「ホーム画面に追加」を選びます。
3. 追加後、ホーム画面のアイコンから起動します。
4. 通知許可を確認し、10秒 / 20秒 / 30秒テスト通知を実行します。
5. オフラインで再起動し、基本画面が開くことを確認します。

## 通知の制約（重要）

- 通知は Android / Chrome / 省電力設定 / バックグラウンド制限に左右されます。
- `setTimeout` を使う長時間後通知は不安定です。
- v0.1 の通知は、ページを開いている間の短時間テストを主目的にしています。
- v0.1 では、ページ再読み込み後の通知タイマー復元は未対応です。
- ブラウザ終了後・端末再起動後の通知保証はありません。
- `localStorage` には基準時刻と選択プリセットを保存しますが、通知タイマーの永続復元は行いません。
- ネイティブアプリのアラームのような厳密保証はありません。
